# Story EP3-S5 — Review e Aprovação
**Epic:** EP3 — Sistema de Landing Pages
**Prioridade:** P2
**Responsável:** @pm
**Status:** Done
**Estimativa:** 0.5 sprint

---

## Contexto

Antes do deploy em produção, o cliente precisa revisar e aprovar a LP em staging. É necessário um processo formal de aprovação para evitar retrabalho.

---

## User Story

**Como** gestor da Escalando Premoldados,
**quero** um processo estruturado de revisão e aprovação de LP com o cliente,
**para que** o go-live seja feito apenas com aprovação explícita.

---

## Acceptance Criteria

- [ ] Checklist de revisão interna (antes de enviar ao cliente): responsivo, formulário, velocidade, copy
- [ ] URL de staging enviada ao cliente com instruções de revisão
- [ ] Formulário ou canal de feedback do cliente (comentários ou WhatsApp)
- [ ] Registro da aprovação (print ou mensagem)
- [ ] SLA: cliente tem 48h para revisar; sem resposta = aprovado

---

## Tarefas Técnicas

- [ ] Criar checklist de QA interno
- [ ] Criar template de mensagem de envio ao cliente
- [ ] Documentar processo no playbook

---

## Definition of Done

- [ ] Checklist interno documentado
- [ ] Pelo menos 1 LP aprovada via processo formal
- [ ] Playbook atualizado
